# State Adoption Checklist

*This draft document is intended for decision makers to see the steps involved in adopting EMMY*

If you’ve understood what Eligibility Made Easy(EMMY) can improve for your applicants, it’s time to see the path from evaluating the tool to integrating EMMY into your application process.

1. # Explore
  

* Read our ‘[What is EMMY](https://confluenceent.cms.gov/spaces/SFIV/pages/1244862980/Income+Verification+as+a+Service+IVaaS+A+Guide+for+Leaders)?’ Guide if you haven’t already
* Explore the [GitHub Repository](https://github.com/DSACMS/iv-cbv-payroll/) for EMMY to see the tool’s source code
* Review current [GitHub Issues](https://github.com/DSACMS/iv-cbv-payroll/issues) to see current areas of active development

1. # Demo
  

* Work with the EMMY team at the contact email here to arrange a demo of the EMMY flowF
* The EMMY team a CMS can also help you beta test EMMY with individual applicants, to get a sense of the user experience

1. # Integrate
  

* Work with the EMMY team on an Authority to Operate (ATO agreement) to run an instance of the EMMY service for your organization
* Define configuration as needed, e.g. setting the ‘look back’ period for Community Engagement verification
* Create on-ramps for applicants to start the Income and/or Community Engagement verification process
* Create API endpoints to receive the verification data returned by EMMY

1. # Launch
  

* Promote your application process to recipients
* Launch pages and connections to EMMY as part of the application process, either online or in-person
* Work with the EMMY team to get access to metrics on the number of visitors, and the numbers who successfully submit their information using EMMY

1. # Update
  

* Your engineering team should follow the [GitHub releases](https://github.com/DSACMS/iv-cbv-payroll/releases) to see if any new features or bug fixes are being released for EMMY
